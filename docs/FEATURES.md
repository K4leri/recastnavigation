# RecastDemo (Zig-порт) — функциональные возможности

Документ описывает реальные возможности демо-приложения `recast_demo` —
интерактивного GUI-визуализатора navmesh, портированного с
`recastnavigation/RecastDemo` на Zig 0.16. Описано только то, что фактически
реализовано в исходниках `demo/src/`.

## 1. Обзор

`recast_demo` — окно 1280×720 на DVUI (glfw + OpenGL 3.3/4.1 core, рендер на
zgl) с собственным 3D-проходом поверх которого рисуется UI. Приложение
позволяет:

- загрузить меш (`.obj`) из каталога ассетов и построить по нему navmesh;
- выбрать один из трёх типов сборки (Solo / Tile / Temp Obstacles);
- настроить все параметры построения через слайдеры в реальном времени и
  пересобрать кнопкой **Build**;
- интерактивно тестировать navmesh-запросы (поиск пути, raycast, расстояние
  до стены, поиск полигонов и т. д.);
- редактировать сцену (off-mesh-связи, выпуклые объёмы-зоны), управлять толпой
  агентов (DetourCrowd), обрезать недостижимые части navmesh;
- редактировать area-типы (цвет, cost, флаги) и poly-флаги в рантайме;
- визуализировать промежуточные структуры Recast (воксели, регионы, контуры,
  poly-mesh, detail-mesh) в 16 режимах отрисовки;
- загружать и прогонять тест-кейсы (`TestCases/*.txt`);
- сохранять/загружать собранный navmesh в бинарный файл (формат MSET).

UI-окна (фиксированы по краям, как в оригинале):

- **Tools** (слева) — выбор активного инструмента и его настройки.
- **Properties** (справа) — выбор сэмпла, входного меша, параметры сборки,
  кнопка Build, debug-режимы отрисовки, переключатели видимости окон.
- **Log** (снизу по центру) — лог сборки (`BuildContext`), со скроллом.
- **Test Cases** — выбор и прогон тест-кейсов.
- **Poly Flags** — менеджер poly-флагов (скрыт по умолчанию).

Каждое окно можно скрыть/показать чекбоксами в секции «Show» панели Properties
(Build Log, Tools Panel, Test Cases, Poly Flags).

Лог сборки (время, число тайлов/полигонов, ошибки) пишется в окно Log.

## 2. Типы сэмплов

Выбираются дропдауном **Sample** в панели Properties: `Solo Mesh`,
`Tile Mesh`, `Temp Obstacles`. Все три строят navmesh по одному и тому же
конвейеру Recast (rasterize → filter → compact → erode → regions → contours →
poly-mesh → detail-mesh → Detour-данные), но различаются стратегией тайлинга.

### Solo Mesh (`sample_solo.zig`)

Вся геометрия растеризуется и строится за один проход в один тайл
(`max_tiles = 1`, `max_polys = 1024`). Поддерживает полный набор из 16 режимов
отрисовки промежуточных структур (см. раздел 7), т. к. хранит все
промежуточные результаты (heightfield, compact, contour set, poly-mesh,
detail-mesh). Это сэмпл по умолчанию при запуске.

### Tile Mesh (`sample_tile.zig`)

Тайловая сборка: сцена делится на квадратные тайлы со стороной **Tile Size**
(вокселей), каждый тайл строится отдельно с расширением границ на
`border_size = walkable_radius + 3`. Размер тайла регулируется слайдером
(16…1024). В UI показываются: число тайлов (W×H), Max Tiles и Max Polys
(вычисляются по битовому распределению, как в оригинале). Чекбокс **Build All
Tiles**. Тайлы добавляются в multi-tile `dtNavMesh`. Режимы отрисовки: Input
Mesh, Navmesh, Navmesh Trans, Navmesh BVTree, **Navmesh Portals** (порталы
между тайлами).

### Temp Obstacles (`sample_temp_obstacles.zig`)

Сборка через `dtTileCache` со слоями (`buildHeightfieldLayers` →
`buildTileCacheLayer`), что позволяет в рантайме добавлять/удалять
**динамические препятствия** (цилиндры) без полной пересборки. Tile Size
16…128. Компрессор слоёв — без сжатия (no-op). В этом режиме панель Tools
показывает не выбор инструмента, а подсказку: `LMB: add  Shift+LMB: remove`.
Препятствия рисуются красными цилиндрами; tilecache обновляется покадрово.
Счётчик препятствий и время сборки — в секции Tile Cache настроек. Чекбокс
**Keep Itermediate Results** (sic, как в коде). Режим отрисовки один —
Navmesh.

## 3. Параметры построения navmesh

Общие настройки (`CommonSettings`, рисуются `drawCommonSettings`) — одинаковы
для всех трёх сэмплов, слайдеры в панели Properties. Порядок и диапазоны 1-в-1
с оригиналом:

**Rasterization**
- Cell Size — 0.10…1.00 (по умолч. 0.3)
- Cell Height — 0.10…1.00 (0.2)
- строка `Voxels W x H` — размер воксельной сетки (только чтение)

**Agent**
- Height — 0.10…5.00 (2.0)
- Radius — 0.00…5.00 (0.6)
- Max Climb — 0.10…5.00 (0.9)
- Max Slope — 0…90° (45)

**Region**
- Min Region Size — 0…150 (8)
- Merged Region Size — 0…150 (20)

**Partitioning** (радиокнопки)
- Watershed (по умолчанию)
- Monotone
- Layers

**Filtering** (чекбоксы)
- Low Hanging Obstacles
- Ledge Spans
- Walkable Low Height Spans

**Polygonization**
- Max Edge Length — 0…50 (12)
- Max Edge Error — 0.1…3.0 (1.3)
- Verts Per Poly — 3…12 (целое, 6)

**Detail Mesh**
- Sample Distance — 0…16 (6)
- Max Sample Error — 0…16 (1)

Кроме общих, у каждого тайлового сэмпла есть **Tile Size** (см. раздел 2).

Сборка запускается кнопкой **Build** в Properties. Время сборки выводится
строкой `Build Time`. Чекбокс **Auto-rebuild on changes** — автоматически
пересобирать navmesh при изменениях, влияющих на запечённые tile-данные
(правки флагов/area-типов). Если выключен — внизу экрана появляется красное
уведомление `! Navmesh rebuild needed — press Build`.

> Примечание: чекбоксы Partitioning и Filtering присутствуют в UI и хранятся в
> настройках, но текущий build-конвейер сэмплов жёстко использует watershed-
> партиционирование и всегда применяет три фильтра (значения чекбоксов не
> разветвляют конвейер).

## 4. Инструменты (Tools)

Активный инструмент выбирается радиокнопками в панели Tools (кроме режима Temp
Obstacles, где инструмент один — добавление препятствий). Доступны:
**Test Navmesh**, **Prune NavMesh**, **Create Off-Mesh Connections**,
**Create Convex Volumes**, **Create Crowds**, **Disabled**.

Общий принцип: левая кнопка мыши (ЛКМ) по сцене делает «клик» в активный
инструмент. Точка клика берётся пересечением луча с мешем (Möller–Trumbore,
лицевые грани), иначе с плоскостью y = 0.

В режиме **Disabled** ЛКМ по navmesh печатает в stderr информацию о полигоне
под точкой (ref, высота, флаги) — инспекция.

### 4.1. Test Navmesh (`tool_navmesh_tester.zig`)

Интерактивное тестирование запросов Detour. ЛКМ ставит **старт** (зелёный),
Shift+ЛКМ — **конец** (красный). Старт/конец рисуются wire-цилиндром агента
(радиус/высота/climb из настроек сэмпла). Режимы (радиокнопки):

- **Pathfind Follow** — `findPath` + сглаживание пути по поверхности
  (`moveAlongSurface` + funnel, буфер до 2048 точек). Рисуется чёрная линия
  сглаженного пути.
- **Pathfind Straight** — `findPath` + `findStraightPath`. Рисуется прямой
  путь (отрезки) + угловые точки, окрашенные по флагу (start/end/off-mesh).
- **Pathfind Sliced** — пошаговый `initSlicedFindPath` / `updateSlicedFindPath`
  (по 20 итераций) / `finalizeSlicedFindPath`, затем straight-path.
- **Distance to Wall** — `findDistanceToWall`: круг радиусом до ближайшей
  стены + нормаль (красный отрезок).
- **Raycast** — `raycast` от старта к концу: отрезок до точки попадания
  (красный — попал в стену, зелёный — дошёл).
- **Find Polys in Circle** — `findPolysAroundCircle` (радиус = расстояние
  старт→конец); найденные полигоны подсвечены, дуги-стрелки к полигону-родителю.
- **Find Polys in Shape** — `findPolysAroundShape` по box-полигону вокруг
  отрезка; подсветка + дуги-родители + контур фигуры.
- **Find Local Neighbourhood** — `findLocalNeighbourhood` (радиус =
  agentRadius × 20); подсветка + дуги + круг.

**Include / Exclude Flags** — две секции чекбоксов (по списку poly-флагов из
реестра): задают `setIncludeFlags` / `setExcludeFlags` фильтра запроса.
Изменение немедленно пересчитывает результат. Внизу — счётчик
`polys: N  waypoints: M`.

### 4.2. Prune NavMesh (`tool_prune.zig`)

Обрезка navmesh до достижимой области (порт `Tool_NavMeshPrune`). ЛКМ —
flood-fill по связности полигонов (обход линков тайла) от точки клика; залитая
область подсвечивается белым. Кнопки:

- **Clear Selection** — сбросить пометку.
- **Prune Unselected** — выставить флаг `disabled` всем НЕ помеченным
  полигонам (они становятся непроходимыми).

### 4.3. Create Off-Mesh Connections (`tool_offmesh.zig`)

Создание off-mesh-связей (прыжки/лестницы). ЛКМ — первая точка (старт),
повторный ЛКМ — вторая точка (конец) → связь добавляется в геометрию (радиус
0.6, area = jump, flag = jump), navmesh пересобирается. Shift+ЛКМ — отмена
текущей незавершённой связи. Радиокнопки **One Way** / **Bidirectional**
задают направленность. Уже созданные связи рисуются всегда (столбики + круги +
дуга со стрелкой) самим сэмплом.

### 4.4. Create Convex Volumes (`tool_convex.zig`)

Создание выпуклых объёмов — зон, красящих область в выбранный area-тип. ЛКМ —
добавить точку контура (по XZ); клик по последней (красной) точке — построить
объём (выпуклая оболочка gift-wrapping всех точек). Shift+ЛКМ — удалить объём
под курсором. После изменения navmesh пересобирается. Слайдеры:

- **Shape Height** (0.1…20) — высота объёма;
- **Shape Descent** (0.1…20) — опускание нижней грани;
- **Poly Offset** (0…10) — расширение контура (`offsetPoly`).

**Area Type** — список area-типов (из реестра) радиокнопками: выбор типа для
следующего объёма. У каждого типа — иконка-карандаш, открывающая
**модальный редактор** area-типа: имя, цветовой пикер (RGB + hex),
чекбоксы reachability-флагов, слайдер cost. OK — применить, Cancel/Esc/закрытие
— отменить. Кнопка **+ Add Area Type** создаёт новый тип через тот же
редактор (черновик; тип создаётся только при OK). Кнопка **Clear Shape**
сбрасывает текущую заготовку.

### 4.5. Create Crowds (`tool_crowd.zig`)

Симуляция толпы агентов через DetourCrowd (до 128 агентов). Режимы
(радиокнопки):

- **Create Agents** — ЛКМ добавить агента, Shift+ЛКМ удалить ближайшего.
  Новый агент сразу получает текущую цель, если она задана.
- **Move Target** — ЛКМ назначить цель пути ВСЕМ агентам (`requestMoveTarget`),
  Shift+ЛКМ — velocity-движение напрямую без pathfinder (`requestMoveVelocity`).
- **Select Agent** — ЛКМ выбрать ближайшего агента (для detail-визуализации).
- **Toggle Polys** — ЛКМ переключить флаг `disabled` у полигона под кликом
  (агенты обходят отключённые).

Управление симуляцией — хоткеями: **SPACE** run/pause, **1** — один шаг.
Статус (`- RUNNING -` / `- PAUSED -`) и счётчик агентов — в панели.

**Options** (дерево, применяются ко всем агентам при изменении):
Optimize Visibility, Optimize Topology, Anticipate Turns, Obstacle Avoidance,
**Avoidance Quality** (0…3 — Low/Medium/Good/High, 4 пресета avoidance),
Separation, **Separation Weight** (0…20).

**Selected Debug Draw** (для выбранного агента, либо для всех при Show Detail
All): Show Corners, Show Collision Segments, Show Path, Show VO (облако
сэмплов obstacle-avoidance), Show Path Optimization, Show Neighbors.

**Debug Draw**: Show Labels (индекс над агентом), Show Prox Grid (прокси-сетка
плотности), Show Nodes (узлы A*-поиска), Show Perf Graph (графики времени
update и числа velocity-сэмплов, 2D-оверлей), Show Detail All.

Агенты рисуются цилиндрами (цвет по target_state), кольцами на земле и сверху,
следом (затухающие линии), стрелками желаемой/реальной скорости. Над целью —
подпись `TARGET`.

## 5. Area types и Poly flags

### Area types (`area_types.zig`)

Рантайм-реестр area-типов (до 64, как `DT_MAX_AREAS`). Каждый тип хранит: имя,
цвет (RGBA), набор poly-флагов (reachability, запекаются в navmesh) и **cost**
(коэффициент стоимости движения для `QueryFilter`). Шесть встроенных типов:

| id | имя    | цвет          | флаги       | cost |
|----|--------|---------------|-------------|------|
| 0  | Ground | (0,192,255)   | walk        | 1.0  |
| 1  | Water  | (0,0,255)     | swim        | 10.0 |
| 2  | Road   | (50,20,12)    | walk        | 1.0  |
| 3  | Door   | (0,255,255)   | walk+door   | 1.0  |
| 4  | Grass  | (0,255,0)     | walk        | 2.0  |
| 5  | Jump   | (255,255,0)   | walk+jump   | 1.5  |

Редактируются через редактор в инструменте Convex Volumes (см. 4.4).
Изменения применяются так:
- **cost** → пушится в фильтры tester/crowd немедленно (на следующем запросе);
- **flags** → запекаются в tile-данные при сборке → требуется rebuild;
- **color** → только debug-draw, мгновенно.

Можно добавлять пользовательские типы (`+ Add Area Type`, авто-цвет из
палитры). Встроенные типы нельзя удалить (имя несменяемо помечено `builtin`).
Неизвестные/мусорные area нормализуются в Ground при сборке; неизвестный area
рисуется красным.

### Poly flags (`poly_flags.zig`)

Рантайм-реестр poly-флагов (битовая маска u16, максимум 16 флагов). Встроенные:
`walk` (0x01), `swim` (0x02), `door` (0x04), `jump` (0x08). Бит 0x10 (`disabled`)
зарезервирован под Toggle Polys / Prune и в пользовательский список не входит.

Менеджер — окно **Poly Flags** (чекбокс Poly Flags в секции Show панели
Properties). Показывает все флаги с их hex-значением, позволяет удалять
пользовательские (`remove`) и добавлять новые (текстовое поле + **Add Flag**).
Флаги используются везде: редактор area-типов и include/exclude-фильтр тестера.

## 6. Управление

### Камера (`camera.zig`)

Модель RecastDemo: орбита + полёт. `view = rotX(pitch)·rotY(yaw)·translate(-pos)`,
перспектива (FOV 50°, near 1, far по размеру сцены).

- **ПКМ (зажать) + движение мыши** — орбита (вращение, pitch ограничен ±90°).
- **Колесо мыши** — зум (шаг масштабируется удалением от центра сцены).
- **W/S** — вперёд/назад, **A/D** — влево/вправо, **Q/E** — вниз/вверх (полёт
  вдоль осей камеры). Скорость пропорциональна удалению от сцены; **Left Shift**
  — ускоренное перемещение.
- **F** или **Home** — сброс камеры по bounding box сцены (reset view).

Управление с клавиатуры работает всегда, кроме случая, когда фокус в текстовом
поле UI (тогда хоткеи подавляются). Ввод камеры/пикинг игнорируются, когда
курсор над панелью UI.

### Хоткеи

- **W/S/A/D/Q/E** — движение камеры; **Left Shift** — ускорение.
- **F / Home** — reset view.
- **ЛКМ** — клик в активный инструмент (пикинг по сцене).
- **Shift+ЛКМ** — альтернативное действие инструмента (конец/удаление/velocity).
- **SPACE** — run/pause толпы (только инструмент Crowd).
- **1** — один шаг симуляции толпы.
- **Esc** — выход (по фронту нажатия; подавлен, когда открыт диалог редактора
  area-типа — там Esc закрывает диалог).
- **C** — переключить режим отсечения граней (off / back / front), печатается в
  stderr.
- **P** — печать текущей камеры в stderr (`--cam=...` для воспроизведения).
- **V** — перебор 8 вариантов рендера вокселей (для отладки), активный вариант
  печатается в stderr.

Курсор корректно масштабируется под HiDPI (процесс делается DPI-unaware на
Windows для попадания кликов).

## 7. Визуализация / debug-draw режимы

Режим отрисовки выбирается радиокнопками в секции **Debug Settings** панели
Properties (доступны только режимы, для которых есть данные). Все промежуточные
структуры Recast визуализируются собственным GL-рендером (`debug_draw_gl.zig`)
с туманом, MSAA, polygon-offset и checker-текстурой пола.

### Solo Mesh — 16 режимов

- **Input Mesh** — исходный меш с раскраской по склону (walkable серый,
  unwalkable подмешан tan) и checker-текстурой.
- **Navmesh** — собранный navmesh (полигоны по area-цвету).
- **Navmesh Trans** — navmesh поверх полупрозрачно (без подложки-меша).
- **Navmesh BVTree** — navmesh + BV-дерево.
- **Navmesh Nodes** — navmesh (+ узлы).
- **Voxels** — solid heightfield (воксели).
- **Walkable Voxels** — проходимые воксели.
- **Compact** — compact heightfield.
- **Compact Distance** — поле расстояний.
- **Compact Regions** — цветные регионы.
- **Region Connections** — регионы + дуги-связи.
- **Raw Contours** — сырые контуры.
- **Both Contours** — сырые + упрощённые.
- **Contours** — упрощённые контуры.
- **Poly Mesh** — полигональная сетка.
- **Poly Mesh Detail** — детальная сетка.

DISABLED-полигоны затемняются (оверлей rgba(0,0,0,128)). Границы меша рисуются
белым wire-боксом; off-mesh-связи и convex-объёмы рисуются всегда независимо от
инструмента. Клавиша **V** даёт 8 вариантов рендера вокселей (mesh/fog/cull/
depth/blend) для отладки.

### Tile Mesh — 5 режимов

Input Mesh, Navmesh, Navmesh Trans, Navmesh BVTree, **Navmesh Portals**.

### Temp Obstacles — 1 режим

Navmesh (+ красные цилиндры препятствий).

## 8. Тест-кейсы (Test Cases)

Окно **Test Cases** (`testcase.zig`) загружает файлы `test_data/TestCases/*.txt`.
Формат строк: `s <sample>`, `f <geom>`, `pf x y z x y z incHex excHex`
(pathfind), `rc ...` (raycast). При выборе теста:

- автоматически переключается тип сэмпла (Solo/Tile/Temp) и входной меш по имени
  из файла (если меш не найден — ошибка в Log, тест не запускается);
- navmesh пересобирается;
- все тесты считаются один раз (кеш): пути (`findPath` + `findStraightPath`) и
  raycast'ы, с замером времени.

В сцене рисуются: маркеры старт/конец, линии путей (голубые), лучи raycast
(зелёный — промах, красный — попадание), подписи `T<index>` над стартами.
В панели — `Sample`, `OK: n/total`, `Total: X ms`.

## 9. Сохранение / загрузка navmesh

В настройках каждого сэмпла есть кнопки **Save** / **Load**. Navmesh
сериализуется в бинарный файл (формат MSET, `navmesh_io.zig`: magic 'MSET',
version 1, заголовок + параметры + тайлы). Пути фиксированы по типу сэмпла:

- Solo → `solo_navmesh.bin`
- Tile → `all_tiles_navmesh.bin`
- Temp Obstacles → `temp_navmesh.bin`

Загрузка заменяет текущий navmesh и сбрасывает инструменты (пересоздают query).

## 10. Аргументы командной строки

Для бенчмаркинга/скриншот-сравнения (`main.zig`):

- `--bench` — камера орбитирует 360° за ~10 с без idle/cap, затем печать средней
  нагрузки (FPS, draw-calls/кадр).
- `--cyclemodes` — перебор всех draw-режимов Solo (поиск крашей).
- `--draw=<имя>` — зафиксировать draw-режим (по имени enum).
- `--cam=pitch,yaw,x,y,z` — задать ракурс камеры.
- `--cull=<N>` — стартовый режим отсечения граней.

## 11. Известные ограничения

- **Имена ASCII-only.** Шрифт UI не содержит кириллицы — имена poly-флагов и
  area-типов задаются латиницей/английским (в коде явно отмечено).
- **Нет персистентности состояния сцены.** Convex-объёмы и off-mesh-связи
  живут только в памяти текущей сессии (нет `.gset`/scene save). Сохраняется/
  загружается лишь собранный navmesh (раздел 9), но не исходная разметка.
- **Партиционирование и фильтры — только UI-чекбоксы.** Конвейер сэмплов всегда
  использует watershed и все три фильтра; значения Partitioning/Filtering в
  build не разветвляются (см. примечание в разделе 3).
- **Solo: фиксированный лимит полигонов** `max_polys = 1024` (один тайл).
- **Temp Obstacles:** компрессор слоёв без сжатия; параметры препятствия
  фиксированы (радиус 1.0, высота 2.0); лимит 128 препятствий.
- **Загрузка меша** — только `.obj` (вершины `v`, грани `f` с триангуляцией
  веером); материалы/нормали из файла игнорируются (нормали считаются сами).
- **Список мешей/тестов** сканируется один раз при старте из каталога ассетов
  (`test_data` рядом с exe/cwd).
